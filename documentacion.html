Descripción del Proyecto

COMEXCARE es un sistema empresarial desarrollado para Comex (empresa mexicana de recubrimientos) que permite la gestión remota de equipos de cómputo (agentes instalados en tiendas a nivel nacional), la distribución y recepción de archivos (actualizaciones de agente, archivos PVSI, DBF), y la generación de reportes de ventas avanzados con múltiples capas de caché y optimización.

El sistema soporta 500+ usuarios concurrentes con tiempos de respuesta <3s incluso procesando más de 7,000 registros, gracias a un sistema de caché escalonado (Redis + tablas materializadas + pre-agregación).

Servicios

DistributionService — Creación y gestión de distribuciones. Envía comandos de descarga a agentes.

AgentUpdateService / ResurtidoAgentUpdateService — Gestión de versiones y despliegue de actualizaciones.

CarteraAbonosCacheService (318L) — Caché optimizado para cartera de abonos.

CarteraAbonosMaterializedService — Consultas mediante vistas materializadas PostgreSQL.

CarteraAbonosUltraFastService — Precarga ultrarrápida (500+ usuarios).

DashboardCacheService — Caché del dashboard con estadísticas.

ReportService (1.026L) — Servicio centralizado de reportes con caché Redis.

Trabajos (Jobs)

ProcessDistributionJob (cola distributions, timeout 300s, 3 reintentos) — Procesa distribución de archivos a destinos.

CarteraAbonosSyncJob (cola default) — Sincroniza caché de cartera de abonos.

CarteraAbonosIncrementalUpdateJob (cola default) — Actualización incremental del caché.

ProcessScheduledDistributions (cola distributions, cada minuto) — Distribuciones programadas.

ProcessScheduledReceptions (cola receptions, cada minuto) — Recepciones programadas.

RetryDistribution (cola retry) — Reintentos de distribuciones fallidas.

Comandos Artisan

computers:check-status — Marca computadoras offline sin heartbeat reciente.

cartera-abonos:sync-cache — Sincroniza caché de cartera de abonos.

db:backup — Respaldo de base de datos.

distribution:restart — Reinicia distribuciones.

queue:start-workers — Inicia workers de cola.

sync:all-cache / sync:cache-full / sync:cache-incremental — Sincronización de caché.

sync:cartera-abonos-cache / sync:club-comex / sync:compras-directo-cache / sync:notas-completas-cache — Sincronización específica.

Tareas Programadas

Cada minuto: ProcessScheduledDistributions, ProcessScheduledReceptions. Cada 5 min: computers:check-status. Dos veces al día: sync:cache-incremental. Diario 11:00: sync de cartera, notas, compras (--last-days=60).

Evento: DistributionProgressUpdated (broadcast en private-distribution.{id} + distributions).

Autorización de canal: DistributionChannel.

Servidor: websocket-server.js (Node.js + ws + Redis pub/sub, puerto 6001). Configuración alternativa: soketi.json.

Cliente: resources/js/bootstrap.js (Laravel Echo + Socket.io).
